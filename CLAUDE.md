# CLAUDE.md — oci-autoscaling

## What This Project Does

Deploys a minimal OCI Instance Pool of Apache web servers behind a flexible
Load Balancer. Each instance displays its own metadata (private IP, instance
OCID, availability domain, shape) on a styled page. Instances are private
(no public IP); outbound access via NAT Gateway.

## Commands

```bash
./apply.sh      # check env, terraform init + apply, then validate
./destroy.sh    # teardown all resources
./validate.sh   # poll LB backends, sample 6 /plain responses
```

## Architecture

Single Terraform phase in `01-autoscaling/`. No modules, no workspaces.

- **Region:** us-ashburn-1
- **Instance:** VM.Standard.A1.Flex (Ampere ARM, 1 OCPU, 1 GB RAM) — cheapest shape
- **LB:** OCI flexible Load Balancer (L7) — round-robin, even distribution
- **Pool:** min 1, initial 4, max 6 across AD-1 and AD-2 (private subnet)
- **Scaling:** CPU threshold policy → scale-out/scale-in rules
- **Startup:** `scripts/userdata.sh` — cloud-init, OCI IMDSv2, apache2

## Scaling Policy

Scale-out triggers when CPU > 60%. Scale-in triggers when CPU < 60%.
The 300s cool-down (OCI minimum) prevents rapid flapping.

| Rule         | Condition | Cool-down | Action      |
|--------------|-----------|-----------|-------------|
| asg-scale-out| CPU > 60% | 300s      | +1 instance |
| asg-scale-in | CPU < 60% | 300s      | -1 instance |

## Validation

`validate.sh` reads LB IP and OCID from Terraform outputs, polls the OCI CLI
for OK backend set status, then samples 6 `/plain` responses. Different private
IPs across responses confirm per-request load balancing is working.

## Key Files

| File | Purpose |
|------|---------|
| `01-autoscaling/instanceconfig.tf` | Instance configuration — shape, image, user data |
| `01-autoscaling/instancepool.tf` | Instance pool, placement, autoscaling policy |
| `01-autoscaling/lb.tf` | Load balancer, backend set, listener, health check |
| `01-autoscaling/networking.tf` | VCN, public/private subnets, IGW, NAT Gateway |
| `01-autoscaling/security.tf` | Security lists — public port 80, private from LB subnet only |
| `01-autoscaling/scripts/userdata.sh` | cloud-init: installs apache2, fetches OCI IMDSv2, writes HTML + /plain |

## OCI Quirks (all baked into userdata.sh)

1. **Network timing:** OCI fires cloud-init before internet routing is ready.
   Script loops on HTTP connectivity to `yum.oracle.com` before running dnf.
2. **Host firewall:** Oracle Linux 9 ships with firewalld active. Script runs
   `firewall-cmd --permanent --add-service=http && firewall-cmd --reload`
   after httpd installs — the security list alone is not enough.

## Auth Pattern

- `~/.oci/config` DEFAULT profile — no credentials in code
- `OCI_COMPARTMENT_ID` env var → `TF_VAR_compartment_ocid` in apply/destroy
- Falls back to tenancy OCID from `~/.oci/config` if env var is unset
- SSH key generated by Terraform (ECDSA P-256), written to `keys/Private_Key`
